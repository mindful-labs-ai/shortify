## 코드 vs 문서 vs 자산

세 가지를 구분한다 — 검색·관리 효율을 위해.

| 종류 | 위치 | 예시 |
|------|------|------|
| **명세 (마크다운)** | `wiki/` | 아키텍처 다이어그램, 디자인 토큰 표, 캐릭터 가이드 |
| **자산 원본** | `/design/`, `/assets/` | Figma/AI 원본, 로고 SVG, 캐릭터 PNG, BGM |
| **코드** | `/src/`, `/src-tauri/`, `/sidecar/` | 실제 구현 |

> `wiki/`에는 **이미지를 직접 두지 않는다**. 필요하면 `/design/`의 export 파일을 상대 경로로 링크.

## 작성 규칙

1. 폴더에는 **반드시 `README.md`(인덱스)**를 둔다.
2. 각 디자인 문서는 상단에 `Owner · Status · Last Updated`를 명시 (디자인 문서 한정).
3. 변경 이력 테이블을 문서 하단에 둔다 (큰 결정만 — 오타 수정은 기록하지 않음).
4. 외부 링크는 가급적 영구 링크(permalink)를 사용.
